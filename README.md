# Why I Built This

Over the years I've collected hundreds of bookmarks to WordPress documentation.

Whenever I wanted to revisit a topic, I found myself jumping between the Developer Handbook, Codex articles, PHP documentation, blog posts, GitHub repositories and Stack Overflow discussions.

While the official documentation is excellent, it's naturally spread across many websites and thousands of individual pages.

I wanted something different.

I wanted a single handbook that I could:

* search quickly
* read offline
* annotate
* upload into Google NotebookLM
* use while preparing for technical interviews
* keep as a permanent reference while working

So I built one.

This repository contains the result.

---

# Project Overview

This handbook currently contains more than **4,800 pages** covering nearly every aspect of modern WordPress backend development.

Topics range from WordPress architecture and plugin development to PHP design patterns, testing, debugging, WP-CLI and modern development workflows.

The repository also contains the scripts used to generate much of the documentation, making the entire process transparent and reproducible.

Although the current focus is backend development, I plan to expand the handbook over time with additional topics such as React, Gutenberg Block Development and other technologies commonly used in modern WordPress projects.

# Perfect for Google NotebookLM

One of the main reasons this handbook exists is because it works extremely well with **Google NotebookLM**.

Simply upload either the complete handbook or a single chapter and NotebookLM can act as a personal WordPress tutor.

It can:

* answer questions
* explain concepts
* generate summaries
* create quizzes
* compare APIs
* explain architecture
* help prepare for interviews

Because the handbook is already organized into focused chapters, it provides excellent source material for AI-assisted learning.

A dedicated guide with example prompts is available in **NOTEBOOKLM.md**.

# Current Limitations

Most of this handbook has been generated using custom scraping scripts that convert the official documentation into printable PDF files.

While every effort has been made to preserve the original content, some pages may contain minor formatting inconsistencies such as:

* spacing
* page breaks
* image placement
* typography
* table formatting

The priority for the initial release was preserving the content.

Improving the presentation and layout is something I plan to work on over time.
